# 新 Client Adapter 验收测试清单

当一个新的 IM 平台（微信、企业微信、飞书或其他）接入 `agent-bridge` 后，用这份清单人工过一遍，确认它符合 [`client-adapter-capabilities.md`](./client-adapter-capabilities.md) 里定义的通用能力。

这份清单不针对某个具体平台，只关注"任何一个新 adapter 都应该具备"的通用表现，不要求懂代码，按步骤操作、对照"预期结果"打勾即可。

## 准备工作

- [ ] `agent-bridge` 已经用新 adapter 的配置启动（`agent-bridge start <channel-name>`），日志显示 adapter 已连接。
- [ ] 有一个可以私聊机器人的账号。
- [ ] 知道怎么看 `agent-bridge` 的运行日志（测试失败时方便截图/贴日志）。

## 测试用例

### 用例 1：基础即时反馈

**操作**：

私聊机器人，输入 `你好`

**通过 ✅**：

5 秒内出现"正在输入 / 正在处理"的信号（打字状态、表情反馈，或一条简短的文字提示，具体形式由平台决定），随后收到正常回复

**不通过 ❌**：

超过 5 秒没有任何反馈；或者反馈出现了但迟迟没有下文

### 用例 2：长任务的进度与最终结果

**操作**：

私聊机器人，输入 `整理今天的 hacknews 热点`，从发送开始持续观察，直到任务结束

**通过 ✅**：

处理过程中出现至少一次"进度类"提示（比如"正在搜索/正在运行某工具"），不是干等；最终收到一条完整、可读的总结报告

**不通过 ❌**：

全程无任何进度提示，超过 30 秒仍无动静；或者最终消息不完整、报错、被截断

### 用例 3：`/new` 开启新会话

**操作**：

先聊几句建立上下文（如 `我叫小明，记住这个名字`），确认机器人有回应后，发送 `/new`，再问 `我叫什么名字？`

**通过 ✅**：

`/new` 后收到"已开启新会话"一类提示；之后机器人不再记得"小明"

**不通过 ❌**：

`/new` 无反馈；或新会话仍记得之前的信息

### 用例 4：中途 `/stop` 能打断任务

**操作**：

发起一个耗时任务（如用例 2 的 hacknews 任务），在出结果前发送 `/stop`

**通过 ✅**：

收到"已停止"一类明确提示，且原任务确实不再继续输出

**不通过 ❌**：

`/stop` 无反应；任务仍在继续输出

### 用例 5：能看懂用户发来的图片/文件

**操作**：

给机器人发一张任意图片或一个小文本文件，并追问一句"这里面是什么内容"

**通过 ✅**：

机器人能正确描述图片内容，或读出文件里的文字内容

**不通过 ❌**：

答非所问、报错，或表现为完全没收到附件

### 用例 6：生成并发送图片和文件给用户

**操作**：

私聊机器人，输入 `用 python 生成一个图片和一个 txt 文件发给我，内容为 "hello from agent"`，观察文字回复之后是否收到附件

**通过 ✅**：

收到一张图片和一个 txt 文件，两个附件都能正常打开；图片里能看到 "hello from agent" 字样，txt 文件内容也是 "hello from agent"

**不通过 ❌**：

只收到文字没有附件；附件打不开；内容和要求的文字不一致

### 用例 7：连续发送多条消息不串话

**操作**：

不等上一条处理完，连续快速发送 3 条不同的简单问题（如"1+1等于几" "2+2等于几" "3+3等于几"）

**通过 ✅**：

三条都被处理，回复顺序与提问顺序一致

**不通过 ❌**：

漏答、回复顺序与问题对应不上
